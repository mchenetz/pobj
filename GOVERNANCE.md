# Governance

## Project Leadership

`entity` is currently maintainer-led.

## Decision Process

- Routine changes: decided in PR review by maintainers.
- Significant changes: require an issue with design rationale and explicit maintainer approval.
- Breaking changes: require migration guidance in docs.

## Maintainers

Current maintainers are listed in `MAINTAINERS.md`.

## Release Process

Releases are cut from tags (`v*`) through GitHub Actions.
